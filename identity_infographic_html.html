先看市场：全球 IAM 玩家，其实活在三种完全不同的逻辑里

同样叫“身份系统”，平台型 IAM、SaaS 型 IAM 与中国本土 IAM 的商业化路径并不相同。

微软 Entra
平台型
默认身份源Office / Azure / Windows 用户天然沉淀到身份层。
商业模式基础免费绑定生态，高级安全与治理能力分层收费。
核心价值控制面入口，而不是单点认证工具。
Okta / Ping
SaaS 型
生态前提依赖 SaaS 标准协议成熟，适合多系统统一登录。
商业模式按用户或企业规模订阅，强调标准化接入。
核心价值让企业跨 SaaS 管理账号、权限与访问。
中国 IAM 厂商
项目型
典型玩家竹云、派拉、Authing、九州云腾等。
商业模式产品 + 咨询 + 集成 + 定制 + 运维。
核心现实更多是安全项目的一环，而不是平台级订阅产品。
全球 IAM = 平台产品；中国 IAM = 更接近安全集成项目。

中国 IAM 为什么始终难以做成“大平台”？

问题不只在产品，而在生态协议、系统结构与采购方式三方面同时受限。

🔌

协议不统一

欧美 SaaS 大量支持 SAML / OIDC / SCIM，接入一个身份系统就能走通很多应用。 而国内大量系统认证方式各自为政，第三方难以标准化集成。

🧩

账号体系碎片化

OA、ERP、CRM、HR、门户常常各自维护账号与权限，IAM 厂商必须按客户逐个对接， 很难形成真正的平台复用与规模效应。

📄

采购逻辑偏项目制

企业通常不会单独立项买“IDaaS”，而是在等保、信创、零信任、大型安全改造项目里顺带采购， 于是身份层更像子模块，而不是独立品类。

微软为什么能成？关键不只是产品，而是它先占住了默认身份源

微软的身份体系不是“卖出去”的，而是在企业软件入口里自然长出来的。

默认身份源

Office 用户
= Entra 用户

企业一旦开始使用 Microsoft 365、Azure、Windows 生态，身份层就自动成为底座。 Entra 的高级版收费，建立在巨大的免费基础用户盘之上。

🎯

Windows、Office、Azure 共同构成企业数字工作入口，身份自然成为底层控制层。

🆓

基础身份免费，条件访问、身份保护、特权管理等能力再分层收费，SKU 逻辑非常清晰。

🌐

生态标准优势

欧美 SaaS 大量支持标准协议，微软身份系统可以轻松外延到更多第三方服务。

🏢

企业习惯延续

从本地 AD 到云端 Entra，用户教育成本低，组织迁移更平滑，历史红利巨大。

为什么飞书、钉钉、Google 都没做到微软这个级别？

不是它们没有身份能力，而是它们没真正占住“企业默认身份源 + 控制面入口”这个位置。

💬

飞书 / 钉钉

它们的身份能力更多嵌在通讯录、组织架构、应用管理中，本质仍服务于协作产品， 还没有强到能脱离协作主产品，单独形成平台税能力。

📧

Google

Google 有 Cloud Identity，但在企业端缺乏像 Windows + AD 那样的长期设备控制与历史基础， 更强的是互联网账号，而不是企业控制面。

🧠

真正差的是位置

微软身份层站在企业 IT 入口之上；其他厂商多数站在单个协作应用之内。 这决定了 Identity 能否被单独商业化。

从商业模型看，能单独卖钱的，其实只有“控制层产品”

如果把企业软件抽象为三层，你会发现 Identity 属于最容易产生平台税的位置。

Control LayerIdentity · Security · Governance
Data LayerOffice · CRM · ERP · 协作应用
Compute LayerCloud · GPU · 基础算力与底层资源
只有当产品位于控制层，才更容易形成“平台税”式独立收费。
